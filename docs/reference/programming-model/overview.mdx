# Inference Workflow

If you are new to SiMa Neat, keep this sequence in mind:

1. Load a compiled model archive (`.tar.gz`) with [`Model`](/reference/{lsa}/classes/simaai-neat-model).
2. Compose a [`Graph`](/reference/{lsa}/classes/simaai-neat-graph) with input nodes, model stages, and output nodes.
3. Build a [`Run`](/reference/{lsa}/classes/simaai-neat-run) in sync or async mode.
4. Push inputs and pull outputs as [`Tensor`](/reference/{lsa}/structs/simaai-neat-tensor) or [`Sample`](/reference/{lsa}/structs/simaai-neat-sample).

## Synchronous inference snippet

Sync: use `run(...)` or `push_and_pull(...)` for request/response style execution.

<CodeTabs>
<CodeTab label="C++" lang="cpp">

```cpp
simaai::neat::Model model("resnet_50_model.tar.gz");
simaai::neat::Graph graph;
graph.add(model.graph());

cv::Mat img = /* your frame (RGB/BGR as configured) */;
auto run = graph.build(img, simaai::neat::RunMode::Sync);
auto out = run.push_and_pull(img, /*timeout_ms=*/1000);
```

</CodeTab>
<CodeTab label="Python" lang="python">

```python
# Pseudo Python sync example.
model = neat.Model("resnet_50_model.tar.gz")

graph = neat.Graph()
graph.add(model.graph())

img = ...  # your frame tensor

run = graph.build(img, neat.RunMode.Sync)
out = run.run(img, timeout_ms=1000)
```

</CodeTab>
</CodeTabs>

## Asynchronous inference snippet

Use async mode when you want to decouple producers and consumers, control
queueing, or overlap IO and compute.
Async: use `push(...)` / `pull(...)` with [`RunOptions`](/reference/{lsa}/structs/simaai-neat-runoptions) to tune queueing and drop behavior.

<CodeTabs>
<CodeTab label="C++" lang="cpp">

```cpp
simaai::neat::Model model("resnet_50_model.tar.gz");
simaai::neat::Graph graph;
graph.add(model.graph());

cv::Mat img = /* your frame */;

simaai::neat::RunOptions opt;
opt.queue_depth = 8;
opt.overflow_policy = simaai::neat::OverflowPolicy::Block;
opt.enable_metrics = true;

auto run = graph.build(img, simaai::neat::RunMode::Async, opt);
run.push(img);
auto out = run.pull(/*timeout_ms=*/1000);
```

</CodeTab>
<CodeTab label="Python" lang="python">

```python
# Pseudo Python async example.
model = neat.Model("resnet_50_model.tar.gz")

graph = neat.Graph()
graph.add(model.graph())

img = ...  # your frame

opt = neat.RunOptions()
opt.queue_depth = 8
opt.overflow_policy = neat.OverflowPolicy.Block

run = graph.build(img, neat.RunMode.Async, opt)
run.push(img)
out = run.pull(timeout_ms=1000)
```

</CodeTab>
</CodeTabs>

## Learn the concepts

- [Model](/reference/programming-model/model): model archive loading and model-driven graph fragments.
- [Graph](/reference/programming-model/graph): assembly, validation, and run/build entry point.
- [Node](/reference/programming-model/node): atomic stages, pre-built groups, and graph boundary nodes.
- [Tensor and Sample](/reference/programming-model/core_types): payload vs metadata envelope.
