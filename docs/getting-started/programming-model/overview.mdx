# Inference Workflow

If you are new to SiMa Neat, keep this sequence in mind:

1. Load a compiled model package (`.tar.gz`) with [`Model`](/reference/{lsa}/classes/simaai-neat-model).
2. Compose a [`Session`](/reference/{lsa}/classes/simaai-neat-session) with input nodes, model stages, and output nodes.
3. Build a [`Run`](/reference/{lsa}/classes/simaai-neat-run) in sync or async mode.
4. Push inputs and pull outputs as [`Tensor`](/reference/{lsa}/structs/simaai-neat-tensor) or [`Sample`](/reference/{lsa}/structs/simaai-neat-sample).

## Synchronous inference snippet

Sync: use `run(...)` or `push_and_pull(...)` for request/response style execution.

<CodeTabs>
<CodeTab label="C++" lang="cpp">

```cpp
simaai::neat::Model model("resnet_50_model.tar.gz");
simaai::neat::Session session;
session.add(model.session());

cv::Mat img = /* your frame (RGB/BGR as configured) */;
auto run = session.build(img, simaai::neat::RunMode::Sync);
auto out = run.push_and_pull(img, /*timeout_ms=*/1000);
```

</CodeTab>
<CodeTab label="Python" lang="python">

```python
# Pseudo Python sync example.
model = neat.Model("resnet_50_model.tar.gz")

session = neat.Session()
session.add(model.session())

img = ...  # your frame tensor

run = session.build(img, neat.RunMode.Sync)
out = run.run(img, timeout_ms=1000)
```

</CodeTab>
</CodeTabs>

## Asynchronous inference snippet

Use async mode when you want to decouple producers and consumers, control
queueing, or overlap IO and compute.
Async: use `push(...)` / `pull(...)` with [`RunOptions`](/reference/{lsa}/structs/simaai-neat-runoptions) to tune queueing and drop behavior.

<CodeTabs>
<CodeTab label="C++" lang="cpp">

```cpp
simaai::neat::Model model("resnet_50_model.tar.gz");
simaai::neat::Session session;
session.add(model.session());

cv::Mat img = /* your frame */;

simaai::neat::RunOptions opt;
opt.queue_depth = 8;
opt.overflow_policy = simaai::neat::OverflowPolicy::Block;
opt.enable_metrics = true;

auto run = session.build(img, simaai::neat::RunMode::Async, opt);
run.push(img);
auto out = run.pull(/*timeout_ms=*/1000);
```

</CodeTab>
<CodeTab label="Python" lang="python">

```python
# Pseudo Python async example.
model = neat.Model("resnet_50_model.tar.gz")

session = neat.Session()
session.add(model.session())

img = ...  # your frame

opt = neat.RunOptions()
opt.queue_depth = 8
opt.overflow_policy = neat.OverflowPolicy.Block

run = session.build(img, neat.RunMode.Async, opt)
run.push(img)
out = run.pull(timeout_ms=1000)
```

</CodeTab>
</CodeTabs>

## Learn the concepts

- [Model](/getting-started/programming-model/model): model pack loading and model-driven pipeline fragments.
- [Session](/getting-started/programming-model/session): assembly, validation, and run/build entry point.
- [Node](/getting-started/programming-model/node): atomic pipeline building block and composition unit.
- [Pipeline](/getting-started/programming-model/pipeline): deterministic node composition and execution handles.
- [Graph](/getting-started/programming-model/graph): hybrid DAG runtime for pipeline + stage composition.
- [Tensor and Sample](/getting-started/programming-model/core_types): payload vs metadata envelope.
- [Input and Output](/getting-started/programming-model/io): sources, sinks, groups, and I/O contracts.
