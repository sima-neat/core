# Tensor and Sample

The runtime data model has two primary types:

- [`Tensor`](/reference/{lsa}/structs/simaai-neat-tensor): typed numeric payload (shape, dtype, layout, storage, device).
- [`Sample`](/reference/{lsa}/structs/simaai-neat-sample): envelope around tensor payload with media/runtime metadata.

## Tensor

Use `Tensor` when you only need numeric data and shape/type semantics.

Typical metadata in tensor:
- shape / dtype / layout
- storage and mapping behavior
- optional image/audio/encoded semantic tags

Related references:
- [`Tensor`](/reference/{lsa}/structs/simaai-neat-tensor)
- [`TensorConstraint`](/reference/{lsa}/structs/simaai-neat-tensorconstraint)
- [`TensorTypes.h`](/reference/{lsa}/files/include-pipeline-tensortypes-h)

### NumPy and PyTorch interop

For Python developers familiar with NumPy and PyTorch, `Tensor` supports DLPack-based interop:

- `Tensor.from_numpy(...)`
- `Tensor.to_numpy(...)`
- `Tensor.from_torch(...)`
- `Tensor.to_torch(...)`
- `Tensor.from_dlpack(...)`
- `Tensor.__dlpack__()`

This keeps interop paths explicit and enables zero-copy where backend data layout permits it.

<CodeTabs>
<CodeTab label="NumPy" lang="python">

```python
import numpy as np
import pyneat as neat

# HWC uint8 image-like tensor
arr = np.random.randint(0, 255, (224, 224, 3), dtype=np.uint8)

t = neat.Tensor.from_numpy(arr, copy=False, image_format=neat.PixelFormat.RGB)
arr_back = t.to_numpy(copy=False)

# If you already have a model, NumPy can be passed directly.
# out = model.run(arr, timeout_ms=2000)
```

</CodeTab>
<CodeTab label="PyTorch" lang="python">

```python
import torch
import pyneat as neat

torch_tensor = torch.randint(0, 255, (224, 224, 3), dtype=torch.uint8)

t = neat.Tensor.from_torch(torch_tensor, copy=False, image_format=neat.PixelFormat.RGB)
torch_back = t.to_torch(copy=False)

# If you already have a model, torch tensors can be passed directly.
# out = model.run(torch_tensor, timeout_ms=2000)
```

## Sample

Use `Sample` when you need pipeline metadata in addition to tensor bytes.

Typical sample fields:
- `caps_string`, `media_type`, `payload_tag`
- `pts_ns`, `dts_ns`, `duration_ns`
- `stream_id`, `frame_id`, `port_name`
- `fields` for bundle outputs

Related references:
- [`Sample`](/reference/{lsa}/structs/simaai-neat-sample)
- [`GraphOptions.h`](/reference/{lsa}/files/include-pipeline-graphoptions-h)
- [`PullError`](/reference/{lsa}/structs/simaai-neat-pullerror)

<CodeTabs>
<CodeTab label="C++" lang="cpp">

```cpp
#include "neat/runtime.h"
#include "neat/nodes.h"

simaai::neat::Graph graph;
graph.add(simaai::neat::nodes::Input({}));
graph.add(simaai::neat::nodes::Output({}));

auto run = graph.build(simaai::neat::Sample{}, simaai::neat::RunMode::Async);

simaai::neat::Sample in;
in.kind = simaai::neat::SampleKind::Tensor;
in.stream_id = "cam-0";
in.frame_id = 42;
// in.tensor = ...;

run.push(in);
auto out = run.pull(1000);
if (out) {
  std::cout << "stream=" << out->stream_id
            << " frame=" << out->frame_id
            << " media_type=" << out->media_type << "\n";
}
```

</CodeTab>
<CodeTab label="Python" lang="python">

```python
import pyneat as neat

graph = neat.Graph()
graph.add(neat.nodes.input(neat.InputOptions()))
graph.add(neat.nodes.output(neat.OutputOptions()))

run = graph.build(neat.Sample(), neat.RunMode.Async)

sample = neat.Sample()
sample.kind = neat.SampleKind.Tensor
sample.stream_id = "cam-0"
sample.frame_id = 42
# sample.tensor = ...

run.push(sample)
out = run.pull(1000)
if out is not None:
  print(f"stream={out.stream_id} frame={out.frame_id} media_type={out.media_type}")
```

</CodeTab>
</CodeTabs>
