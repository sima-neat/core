# Session

[`Session`](/reference/{lsa}/classes/simaai-neat-session) is the runtime entry point.
It owns pipeline assembly, validation, and build/run orchestration.

Think of `Session` as the place where you define pipeline structure once, then execute it many times.

## Core responsibilities

- Add nodes and groups with `add(...)`.
- Inspect pipeline text with `describe()` and `to_gst()`.
- Validate contracts before runtime with `validate(...)`.
- Build a [`Run`](/reference/{lsa}/classes/simaai-neat-run) object via `build(...)`.
- Run inference with `run(...)`.

## Session + Run pattern

<CodeTabs>
<CodeTab label="C++" lang="cpp">

```cpp
simaai::neat::Session session;
session.add(simaai::neat::nodes::Input());
session.add(simaai::neat::nodes::Output());

auto run = session.build(input_tensor, simaai::neat::RunMode::Sync);
auto out = run.push_and_pull(input_tensor, 1000);
```

</CodeTab>
<CodeTab label="Python" lang="python">

```python
# Pseudo Python session + Run example.

session = Session()
session.add(nodes.input())
session.add(nodes.output())


run = session.build(input_tensor, RunMode.Sync)
out = run.push_and_pull(input_tensor, 1000)
```
