Use this page to install or update the Palette Neat Library. You can install it
directly on a DevKit, or inside the Neat Development Environment (referred to as
Neat SDK) when you build from a host machine.

:::note What pairing a DevKit already installs
Pairing a DevKit during `sima-cli sdk setup` installs the Neat Library — and its
bundled PyNeat environment at `~/pyneat` — on the DevKit. The SDK container does
not include the Neat Library; install it in the container only when you
cross-compile C++ apps. Use the steps below to install the Neat Library in the
SDK container, install it on a standalone DevKit, or update an existing install.
:::

## Choose Where to Install

Run the installer from the environment where you build or run Neat applications:

| Environment | Use this when |
| --- | --- |
| Neat SDK with DevKit Sync | You want the host build environment and paired DevKit runtime kept in sync automatically. |
| Neat SDK without DevKit Sync | You only need to build in the container and will transfer binaries to a DevKit manually. |
| DevKit | You want to install or update the runtime directly on the device. |

:::note DevKit Sync keeps the runtime aligned
When you install from inside an SDK container paired with a DevKit through DevKit
Sync, the installer also installs the Neat Library on the paired DevKit. This
keeps the host build environment and the DevKit runtime on the same Neat Library
version.
:::

## Prerequisites

Install `sima-cli` in the environment where you plan to install the Neat Library.
Use the [sima-cli installation guide](/tools/sima-cli/).

Before you begin:

- Make sure the environment running the installer can download SiMa.ai artifacts.
- We recommend creating an account at [community.sima.ai](https://community.sima.ai) to access assets such as precompiled models for testing.

The package installer verifies the package `manifest.json` `platform-version`
before it changes Python, apt, or SDK sysroot packages. In the Neat SDK, it
compares that value with `/etc/sdk-release` `SDK Version`; on a DevKit, it
compares with `/etc/buildinfo` `DISTRO_VERSION`. Development-only
installs on nonstandard images can set `NEAT_INSTALLER_SKIP_PLATFORM_CHECK=ON`
to bypass the guard explicitly.

## Install the Neat Library

Select your environment, then run the steps in order. Install in the **Neat SDK**
container to cross-compile C++ applications; install on the **DevKit** to set up
the runtime there, which also provisions PyNeat at `~/pyneat`. The installer
commands are the same in both places — only the working directory and where you
run them differ.

<CodeTabs>
<CodeTab label="Neat SDK" lang="bash">

Prepare a container-local install directory such as `$HOME/neat`:

<ShellCommand prompt="username@neat-sdk-latest">
cd $HOME && mkdir -p neat && cd neat
</ShellCommand>

Install the default Neat Library package selected by `sima-cli`. You can
optionally install extras, including prebuilt tutorials, source-code tutorials,
and tests:

<ShellCommand prompt="username@neat-sdk-latest">
sima-cli neat install core
</ShellCommand>

To install a specific release instead, append the version to the package name:

<ShellCommand prompt="username@neat-sdk-latest">
sima-cli neat install core@v0.1.0
</ShellCommand>

<ShellCommand prompt="username@neat-sdk-latest">
sima-cli neat install core@v0.2.0
</ShellCommand>

</CodeTab>

<CodeTab label="DevKit" lang="bash">

:::note Update DevKit firmware first
Before installing the Neat Library on a DevKit, update its firmware to a
compatible version with `sima-cli update`. See the
[firmware update guide](https://developer.sima.ai/hardware/getting-started/firmware-update).
:::

Prepare an install directory under `/media/nvme` to avoid filling the root file
system:

<ShellCommand prompt="modalix">
cd /media/nvme && mkdir -p neat && cd neat
</ShellCommand>

Install the default Neat Library package selected by `sima-cli`. You can
optionally install extras, including prebuilt tutorials, source-code tutorials,
and tests:

<ShellCommand prompt="modalix">
sima-cli neat install core
</ShellCommand>

To install a specific release instead, append the version to the package name:

<ShellCommand prompt="modalix">
sima-cli neat install core@v0.1.0
</ShellCommand>

<ShellCommand prompt="modalix">
sima-cli neat install core@v0.2.0
</ShellCommand>

</CodeTab>
</CodeTabs>

## Default PyNeat Environment

Installing the Neat Library on the DevKit also provisions a default PyNeat
virtual environment at `~/pyneat` on the DevKit. This is the supported PyNeat
environment — activate it directly:

<ShellCommand prompt="modalix">
source ~/pyneat/bin/activate
</ShellCommand>

If you run the installer from inside an SDK container paired with DevKit Sync,
the installer provisions PyNeat on the paired DevKit, not only inside the SDK
container. PyNeat is installed into a virtual environment under `~/pyneat` on
the DevKit.

For custom Python virtual environments, see
[Install PyNeat](/getting-started/neat-library/pyneat/).
