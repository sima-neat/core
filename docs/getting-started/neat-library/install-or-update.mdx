Use this page to install or update the Palette Neat Library.

:::tip Neat Library within the Neat SDK
When you pair a DevKit during `sima-cli sdk setup`, the SDK installs its included
Neat Library and bundled PyNeat environment at `~/pyneat` on the DevKit automatically.
:::

## Choose Where to Install

Run the installer from the environment where you build or run Neat applications:

| Environment | Use this when |
| --- | --- |
| Neat SDK | You want to upgrade the Neat Library in the Neat SDK without installing a new SDK version. |
| DevKit | You want to install or update the runtime directly on the device. |

## Prerequisites

Install `sima-cli` in the environment where you plan to install the Neat Library.
Use the [sima-cli installation guide](/tools/sima-cli/).

Before you begin:

- Make sure the environment running the installer can download SiMa.ai artifacts.
- We recommend creating an account at [community.sima.ai](https://community.sima.ai) to access assets such as precompiled models for testing.

## Install the Neat Library

Choose a working directory before running the installer. On the DevKit, install
under `/media/nvme` to avoid filling the root file system. In the SDK container,
use a container-local path such as `$HOME/neat`.

Select your environment, then run the steps in order. Install in the **Neat SDK**
container to cross-compile C++ applications; install on the **DevKit** to set up
the runtime there, which also provisions PyNeat at `~/pyneat`. The installer
commands are the same in both places — only the working directory and where you
run them differ.

<CodeTabs>
<CodeTab label="Neat SDK" lang="bash">

Prepare a container-local install directory such as `$HOME/neat`:

<ShellCommand prompt="username@neat-sdk-latest">
cd $HOME && mkdir -p neat && cd neat
</ShellCommand>

Install the default Neat Library package selected by `sima-cli`. You can
optionally install extras, including prebuilt tutorials, source-code tutorials,
and tests:

<ShellCommand prompt="username@neat-sdk-latest">
sima-cli neat install core@v0.2.2
</ShellCommand>

</CodeTab>

<CodeTab label="DevKit" lang="bash">

:::note Update DevKit firmware first
Before installing the Neat Library on a DevKit, ensure the DevKit software version is compatible with the Neat Library. See the
[Compatibility Guide](/getting-started/compatibility/) for more information.
:::

Prepare an install directory under `/media/nvme` to avoid filling the root file
system:

<ShellCommand prompt="modalix">
cd /media/nvme && mkdir -p neat && cd neat
</ShellCommand>

Install the default Neat Library package selected by `sima-cli`. You can
optionally install extras, including prebuilt tutorials, source-code tutorials,
and tests:

<ShellCommand prompt="modalix">
sima-cli neat install core@v0.2.2
</ShellCommand>

</CodeTab>
</CodeTabs>

## Default PyNeat Environment

Installing the Neat Library on the DevKit also provisions a default PyNeat
virtual environment at `~/pyneat` on the DevKit. This is the supported PyNeat
environment — activate it directly:

<ShellCommand prompt="modalix">
source ~/pyneat/bin/activate
</ShellCommand>

If you run the installer from inside an SDK container paired with DevKit Sync,
the installer provisions PyNeat on the paired DevKit, not only inside the SDK
container. PyNeat is installed into a virtual environment under `~/pyneat` on
the DevKit.

For custom Python virtual environments, see
[Install PyNeat](/getting-started/neat-library/pyneat/).

## Source Builds

Build from source when you need an unreleased change, a custom build
configuration, or you are contributing to the Neat Library. Most users should
use the packaged install above instead.

If you prefer building from source, follow the
[Build](/develop-apps/contribute/build) guide.

For SDK setup and DevKit Sync onboarding, see
[Neat SDK](/getting-started/dev-environment/).
