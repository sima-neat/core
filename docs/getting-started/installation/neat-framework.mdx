Use this page to install or update the SiMa.ai Neat Framework/runtime in either the Neat SDK or directly on a DevKit.

The Neat Framework is installed independently from the Neat SDK. This lets you update the framework/runtime without updating the SDK container.

## Choose Where to Install

Install from the environment where you plan to build or run Neat applications:

| Environment | Use this when |
| --- | --- |
| Neat SDK with DevKit Sync | You want the SDK build environment and paired DevKit runtime kept in sync automatically. |
| Neat SDK without DevKit Sync | You only need to build in the SDK container and will transfer binaries to a DevKit manually. |
| DevKit | You want to install or update the runtime directly on the device. |

:::note DevKit Sync keeps the runtime aligned
When you install from inside a Neat SDK container that has been paired with a DevKit through DevKit Sync, the installer also installs the Neat Framework on the paired DevKit. This keeps the SDK build environment and the DevKit runtime on the same Neat core library version.
:::

## Prerequisites

Install `sima-cli` in the environment where you plan to install Neat. Use the [official guide](https://docs.sima.ai/pages/sima_cli/main.html).

Before you begin:

- Make sure your Modalix DevKit is connected to the Internet.
- We recommend creating an account at [developer.sima.ai](https://developer.sima.ai) to access assets such as precompiled models for testing.

## Prepare an Install Directory

On the DevKit, install under `/media/nvme` to avoid filling the root file system. In the Neat SDK container, use a container-local path such as `$HOME/neat`.

<CodeTabs>
<CodeTab label="Neat SDK" lang="bash">

```bash
cd $HOME && mkdir -p neat && cd neat
```

</CodeTab>

<CodeTab label="DevKit" lang="bash">

```bash
cd /media/nvme && mkdir -p neat && cd neat
```

</CodeTab>
</CodeTabs>

## Install the Latest Stable Build

Use this option for the latest stable build from the `main` branch:

```bash
sima-cli neat install core
```

You can optionally install extras, including prebuilt tutorials, source-code tutorials, and tests.

## Install a Selected Branch or Release

Use this option to try the latest automated build from a specific branch or release. Append `@<branch-or-release>` to the package name — for example, to install the latest `develop` build:

```bash
sima-cli neat install core@develop
```

## After Installation

On the DevKit, installation also provisions a PyNeat virtual environment at `~/pyneat`:

```bash
source ~/pyneat/bin/activate
```

If you run the installer from inside a Neat SDK container paired with DevKit Sync, the PyNeat runtime installation targets the paired DevKit environment. Neat is installed into a virtual environment under `~/pyneat` on the DevKit.

If you prefer building from source, follow the [Build](/contribute/build) guide.

For SDK environment setup and DevKit Sync onboarding, see [Neat SDK](/getting-started/installation/neat-elxr-sdk/).

## Build the PyNeat Wheel for a Custom Environment

The installer provisions the `~/pyneat` virtual environment for you, but you can also build the `pyneat` wheel yourself and install it into an environment of your own (a separate venv, conda, etc.).

1. In a checkout of [`core`](https://github.com/sima-neat/core), build the Python bindings. The wheel is written to `dist/`:

   ```bash
   ./build.sh --python
   ls dist/pyneat-*.whl
   ```

2. Transfer the wheel to the machine with your target environment (for example, copy it onto the DevKit):

   ```bash
   scp dist/pyneat-*.whl <user>@<devkit-host>:~/
   ```

3. Install it into your Python 3.11 environment on that machine:

   ```bash
   python3.11 -m venv ~/my-neat-env
   source ~/my-neat-env/bin/activate
   pip install ~/pyneat-*.whl
   ```

The wheel targets **CPython 3.11** on **`linux_aarch64`**, so build it for — and install it into — a matching environment. For other Python versions and full build options, see [Build](/contribute/build).

## Neat CLI

The installed framework provides the `neat` environment command. Run it from either the Neat SDK or DevKit to see installed component versions, installed `sima-cli` playbooks, and whether newer artifacts are available.

In the Neat SDK, the status output also includes the Insight exposed host ports from `$HOME/.insight-config/neat-port-map.json`.

```bash
neat
```


```bash
neat update
```
