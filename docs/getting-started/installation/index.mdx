As an AI application developer using the SiMa NEAT framework, you have two options to set up your development environment:

1. Install NEAT framework and develop your app directly on a Modalix DevKit.
2. Install NEAT framework and develop in the NEAT eLxr SDK, test on a paired DevKit.

Both paths use the same NEAT framework installation procedure. The difference is where you do your day-to-day development work.

## Prerequisite

- You must have a SiMa.ai Modalix 3.0+ Developer Kit.
- Make sure your Modalix DevKit is connected to the Internet.
- It is recommended that you create an account at [developer.sima.ai](https://developer.sima.ai) to access assets such as precompiled models for testing.
- Install `sima-cli` using the [official guide](https://docs.sima.ai/pages/sima_cli/main.html) on your DevKit and host environment. 

## Which installation should you choose?

|Install On |Best for |Benefits |
| --- | --- | --- |
| DevKit | Convenience and simplicity | <ul><li>Fewer moving parts</li><li>Quick path to first run</li><li>Everything happens on the device</li></ul> |
| NEAT eLxr SDK | Efficiency and faster iteration | <ul><li>Building complex compiled apps in eLxr SDK is generally about 5-10x faster than building directly on DevKit</li><li>SDK helpers integrate the DevKit with the SDK so build/run workflows can stay in the SDK context</li><li>Better fit for agent-driven workflows and rapid iteration</li></ul> |

## Recommended flow

1. Install on [NEAT eLxr SDK](./neat-elxr-sdk.mdx) if you want maximum development efficiency.
2. Use DevKit directly if you want simplicity.

Follow [NEAT Framework](./neat-framework.mdx) to install or update NEAT runtime/framework on either NEAT eLxr SDK or DevKit. The framework installation procedures are common between the two.
