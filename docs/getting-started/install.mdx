## Prerequisite: sima-cli

Install `sima-cli` on your Modalix DevKit before installing NEAT.

Before you begin:
- Make sure your Modalix DevKit is connected to the Internet.
- It is recommended that you create an account at [developer.sima.ai](https://developer.sima.ai) to access assets such as precompiled models for testing.

Install `sima-cli` using the [official guide](https://docs.sima.ai/pages/sima_cli/main.html). After `sima-cli` is ready, choose one NEAT installation path below.
All paths install both C++ and Python NEAT APIs on the Modalix DevKit.

## Official Release

Recommended for most users.

```bash
cd /media/nvme && mkdir -p neat && cd neat && sima-cli install neat
```

You can optionally install extras, including prebuilt and source-code tutorials and tests.

## Latest Build

Use this to try the latest automated build. This installation method also prompts you to select the latest artifacts from any branch.

```bash
wget -O /tmp/install-neat-from-a-branch.sh https://tools.modalix.info/install-neat-from-a-branch.sh && \
  bash /tmp/install-neat-from-a-branch.sh
```

## Specific Build

Use this to test a specific branch and commit.

```bash
wget -O /tmp/install-neat-from-a-branch.sh https://tools.modalix.info/install-neat-from-a-branch.sh && \
  bash /tmp/install-neat-from-a-branch.sh {branch} {githash}
```

Replace:
- `{branch}` with your branch name
- `{githash}` with the 7-character Git commit hash

If you prefer building from source, follow the [build topic](./build.md).

The installation also provisions PyNeat in the `/media/nvme/pyneat` virtual environment. On devices that do not have a NVMe drive, pyneat is installed into the `~/pyneat` virtual environment.

## eLxr SDK Note

If you want to install NEAT inside the eLxr SDK sysroot, you can follow the same installation flow and artifact selection described above.

If you build applications in eLxr SDK and run them on a Modalix DevKit, make sure the DevKit has the same NEAT version installed as the one used in the SDK/sysroot build.

Next, continue with [Hello SiMa](./minimal_example.md) to validate your setup with a minimal app.
