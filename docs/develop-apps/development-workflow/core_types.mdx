# Tensor and Sample

Bytes are not meaning. Metadata is meaning.

Neat moves runtime data with two public types:

- [`Tensor`](/reference/{lsa}/structs/simaai-neat-tensor): numeric payload with shape, dtype, layout, storage, and optional semantic metadata.
- [`Sample`](/reference/{lsa}/structs/simaai-neat-sample): an envelope around one or more tensors plus stream, frame, timestamp, payload, and route metadata.

Use `Tensor` when you only need model input or output data. Use `Sample` when the graph must preserve stream identity, timestamps, encoded media caps, or multi-output structure.

## Choose TensorList or Sample

| Task | C++ input shape | Python input shape | Returns |
| --- | --- | --- | --- |
| Run a model with tensors | `model.run(TensorList{tensor})` | `model.run([tensor])` | `TensorList` / Python list-like tensor collection |
| Run a model with metadata | `model.run(sample)` | `model.run([sample])` | `Sample` |
| Push tensors into a `Run` | `run.push(TensorList{tensor})` | `run.push([tensor])` | `bool` |
| Push samples into a `Run` | `run.push(sample)` | `run.push([sample])` | `bool` |
| Pull tensor-only output | `run.pull_tensors(...)` | `run.pull_tensors(...)` | `TensorList` / Python list-like tensor collection |
| Pull metadata-preserving output | `run.pull(...)` or `run.pull_samples(...)` | `run.pull(...)` or `run.pull_samples(...)` | `Sample` |

In Python, always pass model or graph inputs as a list or tuple. `[tensor]` means “one input.” It does not create a batch dimension.

Batch size belongs to the tensor shape and the compiled model contract. Inspect `model.input_specs()` and `model.compiled_batch_size()` before you treat an input list as a batch.

## Create tensors in Python

Use the constructor that matches your source data.

| Source data | Use |
| --- | --- |
| NumPy array | `pyneat.Tensor.from_numpy(array, copy=True, layout=..., image_format=...)` |
| Torch tensor | `pyneat.Tensor.from_torch(tensor, copy=True, layout=..., image_format=...)` |
| DLPack-compatible object | `pyneat.Tensor.from_dlpack(obj, copy=True, layout=...)` |
| Python list | `pyneat.Tensor.from_list(data, shape=..., dtype=...)` |
| Encoded media bytes | `pyneat.Sample.from_encoded(data, caps_string, port_name=...)` |

Use `copy=True` when the source buffer might go away before Neat finishes reading it. Use `copy=False` only when your app owns the lifetime. No mystery buffers.

<CodeTabs>
<CodeTab label="NumPy" lang="python">

```python
import numpy as np
import pyneat

frame = np.zeros((480, 640, 3), dtype=np.uint8)

tensor = pyneat.Tensor.from_numpy(
    frame,
    copy=True,
    image_format=pyneat.PixelFormat.BGR,
)

outputs = model.run([tensor], timeout_ms=2000)
```

</CodeTab>
<CodeTab label="PyTorch" lang="python">

```python
import torch
import pyneat

torch_tensor = torch.zeros((480, 640, 3), dtype=torch.uint8)

tensor = pyneat.Tensor.from_torch(
    torch_tensor,
    copy=True,
    image_format=pyneat.PixelFormat.RGB,
)

outputs = model.run([tensor], timeout_ms=2000)
```

</CodeTab>
</CodeTabs>

Convert outputs when your app needs framework-native data:

```python
array = outputs[0].to_numpy(copy=True)
torch_tensor = outputs[0].to_torch(copy=True)
```

Most first-run code should not pick memory placement manually. If you need to, Python exposes `TensorMemory.Auto`, `CPU`, `A65`, `EV74`, and `MLA` through the `memory=` argument.

Use `byte_format=` only for byte-format tensor payloads. Do not combine `byte_format=` with `image_format=`.

### Create encoded samples in Python

Use `Sample.from_encoded(...)` when the graph input expects encoded media, such as H.264, and the app owns the bytes.

```python
sample = pyneat.Sample.from_encoded(
    data,
    caps_string="video/x-h264,stream-format=byte-stream,alignment=au",
    pts_ns=pts_ns,
    duration_ns=duration_ns,
    port_name="video",
)

run.push("video", [sample])
```

The `caps_string` tells the graph what the bytes mean. `port_name` ties the sample to the graph input name. If you have decoded pixels, do not use this path; create an image tensor or image sample instead.

## Create tensors in C++

Use `TensorList` for tensor-only input. A `TensorList` contains one tensor per model or graph ingress.

```cpp
simaai::neat::Tensor input = simaai::neat::Tensor::from_cv_mat(
    frame,
    simaai::neat::ImageSpec::PixelFormat::BGR,
    simaai::neat::TensorMemory::CPU);

simaai::neat::TensorList outputs = model.run(
    simaai::neat::TensorList{input},
    /*timeout_ms=*/2000);
```

For multi-input models, pass one tensor per input in the same order as `model.input_specs()` unless the graph exposes named inputs.

### Use OpenCV images in C++

OpenCV images are usually BGR. Say that out loud in the code.

For direct `Model` runs, the explicit path is to convert the `cv::Mat` to a `Tensor` and pass a `TensorList`:

```cpp
cv::Mat image = cv::imread("image.jpg");

auto input = simaai::neat::Tensor::from_cv_mat(
    image,
    simaai::neat::ImageSpec::PixelFormat::BGR,
    simaai::neat::TensorMemory::CPU);

auto outputs = model.run(
    simaai::neat::TensorList{input},
    /*timeout_ms=*/2000);
```

When Neat is built with OpenCV support, `Model`, `Graph`, and `Run` also expose `std::vector<cv::Mat>` convenience overloads. Pass one `cv::Mat` per ingress:

```cpp
auto model_outputs = model.run(std::vector<cv::Mat>{image});
auto graph_outputs = graph.run(std::vector<cv::Mat>{image});

auto run = graph.build();
run.push("image", std::vector<cv::Mat>{image});
```

Prefer `Tensor::from_cv_mat(...)` when the example must state pixel format, memory, or lifetime explicitly. Use the OpenCV convenience overload when the graph/model options already describe the input contract.

## Read tensor contracts

Inspect specs before you allocate inputs or decode outputs.

```python
for spec in model.input_specs():
    print(spec.shape, spec.dtype, spec.layout)

for spec in model.output_specs():
    print(spec.shape, spec.dtype, spec.layout)
```

A tensor contract can include:

- shape and byte size;
- `TensorDType`, such as integer, float, or BF16 payloads;
- `TensorLayout`, such as HWC or model-specific tensor layout;
- image pixel format through image semantic metadata;
- quantization, tessellation, detection, text, audio, byte-stream, encoded, or preprocess semantic metadata.

Do not assume every public boundary is FP32. Read the spec.

## Decode model output

Start with raw tensors. Decode only when the output contract says the payload is a detection, pose, or segmentation format.

| Output contract | Use | Result |
| --- | --- | --- |
| Raw tensor | Use the returned tensor directly, or convert with `to_numpy(...)` / `to_torch(...)` | Tensor data in the model output dtype and shape |
| Packed boxes as tensors | `pyneat.decode_bbox(outputs, clamp_to=(width, height), top_k=..., strict=...)` | One float32 `[N, 6]` tensor per input: `x1`, `y1`, `x2`, `y2`, `score`, `class_id` |
| One packed box tensor as typed records | `pyneat.detections.decode_bbox_tensor(tensor, width, height, expected_topk=..., strict=...)` | `BoxDecodeResult` with `boxes` records and retained raw bytes |
| Packed pose | `pyneat.decode_pose(outputs, clamp_to=(width, height), top_k=..., strict=...)` | Boxes plus keypoints |
| Packed segmentation | `pyneat.decode_segmentation(outputs, clamp_to=(width, height), top_k=..., strict=...)` | Boxes plus masks |

```python
outputs = model.run([tensor], timeout_ms=2000)
boxes = pyneat.decode_bbox(outputs, clamp_to=(image_width, image_height), strict=True)
box_array = boxes[0].to_numpy(copy=True)
```

Use decode helpers only on the output formats they expect.

## Use Sample for metadata

Use `Sample` when payload data needs identity or timing.

Common constructors and helpers:

| Need | C++ | Python |
| --- | --- | --- |
| Wrap a tensor with a port name | `make_tensor_sample("image", tensor)` | `pyneat.make_tensor_sample("image", tensor)` |
| Wrap an OpenCV image | `make_image_sample(image, PixelFormat::BGR, TensorMemory::CPU)` | Create a `Tensor` with `Tensor.from_numpy(...)`, then call `make_tensor_sample(...)` |
| Wrap encoded media bytes | `make_encoded_sample(bytes, caps_string, pts_ns, dts_ns, duration_ns)` | `pyneat.make_encoded_sample(...)` or `pyneat.Sample.from_encoded(data, caps_string, port_name=...)` |
| Wrap text | `Tensor::from_text(...)` plus `make_tensor_sample(...)` | `pyneat.make_text_sample("text", text)` |
| Wrap several logical fields | `make_bundle_sample({field0, field1})` | Create `pyneat.Sample()` and call `append(...)` |

Common fields:

| Field | Use |
| --- | --- |
| `kind` | Tells whether the payload is `Tensor`, `TensorSet`, `Bundle`, or `Unknown`. |
| `tensor` | Single tensor payload when `kind == Tensor`. |
| `tensors` | Multiple tensors at one logical output when `kind == TensorSet`. |
| `fields` | Nested samples when `kind == Bundle`. |
| `stream_id` | Stable stream identity for multistream graphs. |
| `frame_id` | Source frame identity for combine, render, and metrics. |
| `pts_ns`, `dts_ns`, `duration_ns` | Timing metadata. |
| `port_name` | Input or output port identity. |
| `payload_type`, `media_type`, `payload_tag`, `caps_string` | Payload family and media/caps metadata. |
| `logical_output_index` | Logical output identity for models with multiple outputs. |

Python `Sample` is iterable. A non-bundle sample behaves like one logical item; a bundle iterates over its `fields`.

### Choose `TensorSet` or `Bundle`

`TensorSet` and `Bundle` both carry more than one payload, but they mean different things:

| Shape | Use it for | Read from |
| --- | --- | --- |
| `TensorSet` | Several tensors that make up one logical input or output, such as boxes plus masks at one output port. | `sample.tensors` |
| `Bundle` | Several logical fields traveling together, such as an image field plus a metadata field. | `sample.fields`, iteration, or `sample[index]` |

Use `TensorSet` when downstream code should treat the tensors as one result. Use `Bundle` when downstream code should route or inspect fields separately. If you mix those up, the bytes may still move, but the meaning gets mugged in an alley.

<CodeTabs>
<CodeTab label="C++" lang="cpp">

```cpp
auto image = simaai::neat::make_tensor_sample("image", image_tensor);
auto metadata = simaai::neat::make_tensor_sample(
    "metadata",
    simaai::neat::Tensor::from_text(metadata_json));

auto bundle = simaai::neat::make_bundle_sample({image, metadata});
bundle.stream_id = "camera-0";
bundle.frame_id = frame_number;

run.push("request", bundle);
```

</CodeTab>
<CodeTab label="Python" lang="python">

```python
image = pyneat.make_tensor_sample("image", image_tensor)
metadata = pyneat.make_text_sample("metadata", metadata_json)

bundle = pyneat.Sample()
bundle.append(image)
bundle.append(metadata)
bundle.stream_id = "camera-0"
bundle.frame_id = frame_number

run.push("request", [bundle])
```

</CodeTab>
</CodeTabs>

When you receive a `TensorSet`, iterate over `sample.tensors`. When you receive a `Bundle`, iterate over the sample itself or over `sample.fields`. Keep the shape honest; future you is the downstream consumer.

</CodeTab>

If a multistream graph has no `stream_id`, it cannot report fair per-stream throughput. Give every stream a name before you tune it.

### Read pulled samples

```python
sample = run.pull("detections", timeout_ms=2000)

if sample is not None:
    print(sample.kind, sample.stream_id, sample.frame_id)
    if sample.kind == pyneat.SampleKind.TensorSet:
        for tensor in sample.tensors:
            print(tensor.shape, tensor.dtype)
```

## Understand payload families

`PayloadType` tells Neat what kind of data crosses a graph boundary.

| Payload family | Use / inspect |
| --- | --- |
| `PayloadType.Image` | Decoded pixels. Inspect image format, width, height, and pixel metadata. |
| `PayloadType.Tensor` | Model or application tensors. Inspect dtype, shape, layout, and semantic metadata. |
| `PayloadType.Encoded` | Encoded media such as H.264. Inspect `caps_string`, codec, timestamps, and payload tag. |
| `PayloadType.Auto` | Let Neat infer from tensor/sample metadata when possible. |

Text, audio, byte-stream, and opaque bytes travel as tensor semantics or specialized specs, not separate `PayloadType` enum values.

For explicit input boundaries, use `InputOptions` with `payload_type`, `format`, size fields, dynamic limits, fps, caps override, live/timestamp behavior, and memory policy. In Python, set format fields with `pyneat.Format` / `pyneat.FormatTag`, not raw strings.

## Advanced type wayfinding

You do not need these types for first inference, but they are useful when you are validating contracts or collecting diagnostics.

| Type | Use it when |
| --- | --- |
| `TensorConstraint` / `TensorSpec` | You need to check dtype, rank, shape, device, image format, or segment constraints before sending a tensor. |
| `DeviceType` | You need to inspect whether a tensor is associated with CPU, APU, CVU, MLA, or an unknown device. |
| `StorageKind` | You need to understand whether storage is CPU-owned, external CPU memory, a GStreamer sample, or a device handle. |
| `PlaneRole` | You are inspecting multi-plane media such as Y, U, V, or UV planes. |
| `BusMessage` | You are reading `GraphReport.bus` after a runtime or build failure. |
| `BoundaryFlowStats` | You are checking which graph boundary saw buffers during a diagnostics run. |
| `MeasureOptions` / `MeasureReport` | You are measuring a `Run` window and need counters, latency, optional plugin timing, or optional power. |

Use these as evidence. Do not turn them into knobs unless the task really needs them.
